# Learn by Doing: Build a Knowledge Graph using Ontology from the Publishing domain 

## Summary 
Create an interactive mini-lesson on what a Knowledge Graph is and who uses them for what. Learn the definition of important terms like ontology and use a familiar domain, the publishing industry (I'm thinking mainly books here). This choice also allows for an exploration of role of Schema.org in driving what we experience in the web. Explore the similarities between having an ontology (a pretty intuitive concept, really, with an unintuitive name) and modern-day data modeling (with a historical eye). Keep it streamlined and beginner friendly.

### Fundamental Attributes 
- Structural: Entities (nodes), Relationships (edges), Properties (of nodes, possibly edges), Directionality of relationships
- Semantics: KG rely on defined ontologies that give specific, contextual meaning to entities and relationships
- Integrates data from external sources, Tracks where each piece of data comes from and reliability, Scalable
- Support search queries 
- Support inference (deriving new insights from existing data)
